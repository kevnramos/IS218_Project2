Git

The Basic Git commands

Initializing a repository

First we want to initialize out git repository on our local machine

Make sure you are in the folder in which you want to initialize the repository

git init

Adding files to stage them for commit

Before you make your first commit you can check what files have been changed and can be added

To do so, run:

git status

This should show the files that have not been staged for commit

To add all of your files so you can commit them, run:

git add .

If you run git status again, you should see a message like 

â git status
On branch master
nothing to commit, working tree clean

Committing your files

Now we want to commit our files to basically save our changes in this time machine we call git

git commit -m "Our first commit"

Now our commit is saved to git

If we want to see all our commits, we can do: git log and we'll see something like this 

â git log
commit 12345678987654321 (HEAD -> master)
Author: FirstName LastName <youremail@email.com>
Date:   Tue Sep 14 13:52:06 2021 -0400

    Our first commit

Making Branches

To make a branch and switch to that branch, you can do:

git branch dev_test
git checkout dev_test

Or you can do a shortcut and do this

git checkout -b dev_test

Pushing to a remote repository

Once you have added and committed your files, you can push them to a remote repository

git push origin/main
